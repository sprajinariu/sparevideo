# Block 4: Connected-Component Labeling (CCL)

**Parent:** [motion-pipeline-improvements.md](motion-pipeline-improvements.md) — see block 4 for the original scope outline.
**Status:** design — not yet implemented.
**Date:** 2026-04-20

---

## Motivation

The current motion pipeline uses `axis_bbox_reduce` to compute a **single** global bounding box from the 1-bit motion mask. Two people walking in opposite corners of the frame collapse into one huge rectangle that spans the whole scene, because the reducer has no concept of distinct objects.

CCL replaces the single-bbox reducer with a streaming connected-component labeler that assigns distinct labels to separate foreground regions, accumulates a per-label bbox, and emits up to `N_OUT` bounding boxes per frame. Downstream, `axis_overlay_bbox` draws one rectangle per surviving bbox.

This is also the first block in the motion pipeline to introduce **union-find with path compression** and an **EOF multi-cycle FSM** running during vblank — architectural patterns not present in any prior stage of the project.

---

## Overview

### What changes

- **New module `axis_ccl`** in `hw/ip/motion/rtl/`, replacing `axis_bbox_reduce` at the same pipeline position. `axis_bbox_reduce` is deleted from the repo (hard replace).
- **`axis_overlay_bbox`** is modified to accept an `N_OUT`-wide bbox array and perform N-wide hit-testing via a `generate for` block. AXIS video passthrough logic unchanged.
- **`sparevideo_top`** rewires the sideband bus from `{min_x, max_x, min_y, max_y, empty}` to packed arrays of `N_OUT` entries. Adds a 2-to-1 mux on the overlay's video input for the new `ccl_bbox` debug ctrl_flow.
- **`sparevideo_pkg`** adds `CTRL_FLOW_CCL_BBOX = 2'b11`.
- **Python models**: new `py/models/ccl.py` (streaming reference — same algorithm as the RTL), new `py/models/ccl_bbox.py` (mask-as-grey + bboxes render composition).
- **TB blanking** bumped to `V_BLANK ≥ 16` lines to absorb worst-case EOF FSM cycle budget.

### Architecture at a glance

```
(current)  msk ──► axis_bbox_reduce ──► 1 × {bbox coords}        ──► u_overlay
(new)      msk ──► axis_ccl         ──► N_OUT × {bbox coords + valid} ──► u_overlay
```

Inside `axis_ccl`:

```
mask in ──► [per-pixel labeling pipeline]      ──► (1-bit AXIS sink, tready=1)
                    │
                    ├─► label line buffer (320 × 6b, 1R1W)
                    ├─► equiv table (64 × 6b, 1R1W)
                    └─► accumulator bank (64 × ~51b, 1R1W)
                                    │
                              on EOF pulse
                                    ▼
            [EOF resolution FSM — phases A/B/C/D during vblank]
                                    │
                                    ▼
            [output bbox double-buffered register file] ──► overlay sideband
```

## Per-pixel datapath

For each foreground mask pixel at `(r, c)`, examine four already-labeled 8-connected neighbors:

```
 NW  N  NE
  W  *
```

- `{NW, N, NE}` are read from the **label line buffer** (previous row), using a 3-deep shift register on the registered line-buffer read output to expose all three positions on the same cycle. The buffer is read one column ahead of the current pixel so NE is available in time.
- `W` is a single 6-bit register holding the label assigned to the immediately previous column in the current row.

Edge handling (no special storage):
- Row 0: all "above" reads return 0 (background), via a `row_idx == 0` mux on the line-buffer output.
- Column 0 and `H_ACTIVE - 1`: off-image neighbors are forced to 0 via column-counter muxes.

### Label decision (combinational)

```
distinct_nonzero = unique foreground labels among {NW, N, NE, W}
// INVARIANT (proven below): |distinct_nonzero| ≤ 2

case |distinct_nonzero|:
  0: label = allocate_new()          // from next_free_counter
  1: label = distinct_nonzero[0]     // inherit
  2: label = min(distinct_nonzero)   // smaller wins
     equiv_wr_en   = 1
     equiv_wr_addr = max(distinct_nonzero)
     equiv_wr_data = min(distinct_nonzero)
```

### Invariant: ≤ 2 distinct labels per pixel

In 8-connected raster CCL, the neighbor set `{NW, N, NE, W}` can contain **at most 2 distinct foreground labels**.

Sketch: `NW` and `N` are adjacent in the previous row — had both been foreground, they would have merged when `N` was labeled. Likewise `N` and `NE`. So `{NW, N, NE}` contributes at most one distinct foreground label. Combined with `W`, that's ≤ 2 distinct. The `W`-vs-`{NW, N, NE}` disjointness case is the canonical "two blobs meeting at the current pixel" scenario.

Consequence: the equivalence table needs at most **one write per pixel**. A single 1W port suffices.

### Allocation & overflow

`next_free_counter` starts at 1 (label 0 is reserved as background and overflow catch-all) and increments on every new allocation. When it reaches `N_LABELS_INT - 1`, further allocations return label 0. All overflow pixels pool into label 0's accumulator — graceful degradation, no corruption of real blob bboxes.

### Per-pixel writes

| Memory | Write | Notes |
|---|---|---|
| Label line buffer | at column `c`, value = assigned label (0 for background pixels) | 1W |
| Equivalence table | at most 1 write (on merge): `equiv[max_label] = min_label` | 1W, gated by merge condition |
| Accumulator bank | RMW at assigned label: `{min_x, max_x, min_y, max_y, count}` updated against `(c, r)` | 1R1W |

Internal pipeline: ~2 stages (line-buffer registered read → window register → decision + writes). Never stalls; `tready` held high.

## EOF resolution FSM

Triggered when the last active mask pixel of the frame is accepted. Runs during vblank while input `tvalid` is 0. Four sequential phases.

### Phase A — Path compression

Walk labels `1 .. N_LABELS_INT - 1`. For each label `L`, chase `equiv[]` pointers until a root is found (`equiv[root] == root`), then second-pass-compress so `equiv[L] = root` directly. Bounded by `MAX_CHAIN_DEPTH` per chase to guarantee a cycle budget.

Cost: `N_LABELS_INT × 2 × max(observed_depth, MAX_CHAIN_DEPTH)` cycles.

### Phase B — Accumulator fold

Single pass over labels. For each non-root label `L` where `equiv[L] != L`:
- Read `acc[L]` and `acc[equiv[L]]`.
- Write merged record to `acc[equiv[L]]`: componentwise `min`/`max`, sum of `count`.
- Clear `acc[L]` (count = 0 marks it consumed).

Cost: `N_LABELS_INT × 2` cycles (1R1W bank, two cycles per fold).

### Phase C — Min-size filter + `N_OUT` selection

Walk `acc[]` one more pass. Skip folded entries (count = 0) and entries with `count < MIN_COMPONENT_PIXELS`. Among survivors, pick the top `N_OUT` by `count` via a simple `N_OUT`-pass max-find.

Cost: `N_OUT × N_LABELS_INT` cycles.

### Phase D — Reset for next frame

- `equiv[L] = L` for all L (identity).
- `acc[L]` reset to sentinel values (`min_x = H-1`, `max_x = 0`, `min_y = V-1`, `max_y = 0`, `count = 0`).
- `next_free_counter = 1`.
- Label line buffer needs no clearing (overwritten on row 0; edge mux returns 0 for row-0 reads).

Cost: `N_LABELS_INT` cycles.

### Cycle budget (at defaults: N_LABELS_INT=64, N_OUT=8, MAX_CHAIN_DEPTH=8)

| Phase | Typical | Worst case (bounded) |
|---|---|---|
| A | ~400 | ~1,024 |
| B | ~130 | ~130 |
| C | ~510 | ~510 |
| D | 64 | 64 |
| **Total** | **~1,100** | **~1,700** |

### Vblank headroom

- **Real VGA 640×480 @ 60 Hz:** ~144,000 DSP cycles of vblank. ~85× headroom. ✅
- **TB reduced blanking (V=16 lines proposed):** ~21,500 DSP cycles. ~12× headroom. ✅

The current TB default of V=6 lines (~8,000 DSP cycles) is too tight for pre-bounding worst case and is bumped to V=16 as part of this plan.

### Output handshake

A single `bbox_swap` pulse at Phase D exit latches the new bbox array into the output double-buffer. The overlay reads the "front" buffer during the next frame while the next frame's CCL writes accumulate into the internal bank — no ping-pong of the accumulator bank is needed yet (see Future Work).

## Overlay changes (`axis_overlay_bbox`)

Single focused change: the hit-test becomes N-wide via `generate`:

```systemverilog
genvar k;
generate
    for (k = 0; k < N_OUT; k = k + 1) begin : g_hit
        assign bbox_hit[k] = bbox_valid_i[k]
                          && on_border(col, row,
                                       bbox_min_x_i[k], bbox_max_x_i[k],
                                       bbox_min_y_i[k], bbox_max_y_i[k]);
    end
endgenerate
assign on_rect = |bbox_hit;
```

`on_border(...)` is the existing per-rectangle border computation, inlined or extracted as a function. AXIS passthrough, pixel-replace logic, and first-frame suppression gating are unchanged.

## `ctrl_flow=ccl_bbox` debug mode

A fourth `ctrl_flow` value for direct visibility into CCL output.

| Value | Name | Output |
|---|---|---|
| `2'b00` | passthrough | raw RGB |
| `2'b01` | motion | RGB + bboxes |
| `2'b10` | mask | B/W mask |
| `2'b11` | **ccl_bbox** | grey-mask canvas + green bboxes |

RTL change, confined to `sparevideo_top`: a 2-to-1 mux selects the overlay's video input between the RGB stream (for `motion`) and a combinational `mask_as_grey_rgb = mask_bit ? 24'h808080 : 24'h202020` (for `ccl_bbox`). CCL and the overlay are otherwise untouched.

Harness: add `ccl_bbox` to `CTRL_FLOW` whitelist and dispatch. Python model composes existing mask + CCL outputs into a grey-canvas render.

## Memory summary

| Memory | Size | Implementation |
|---|---|---|
| Label line buffer | 320 × 6 = 240 B | Distributed LUT-RAM, 1R1W |
| Equivalence table | 64 × 6 = 48 B | Distributed LUT-RAM, 1R1W |
| Accumulator bank | 64 × 51 ≈ 3.3 Kb | Distributed LUT-RAM, 1R1W |
| Output bbox double-buffer | 2 × 8 × 35 ≈ 560 b | Flip-flops (9+9+8+8+1 per entry) |

Total: ~4 Kb distributed RAM + ~600 FFs for output. No BRAM. No interaction with the shared `ram`.

## Verification

### Python reference model — `py/models/ccl.py`

Re-implements the **same** streaming union-find + EOF-fold algorithm as the RTL, not `scipy.ndimage.label` or OpenCV. A trusted oracle using a different algorithm would let a shared spec bug pass both sides silently.

Additionally, `py/tests/test_models.py` cross-checks our reference CCL against `scipy.ndimage.label` on synthetic masks, asserting the **set** of connected components and their bboxes agree independent of label ordering. This guards against algorithm-level divergence.

### Unit testbench — `hw/ip/motion/tb/tb_axis_ccl.sv`

Following the `tb_axis_*` pattern. Minimum coverage:

- **Single blob:** solid rectangle, hollow rectangle, diagonal line.
- **Disjoint blobs:** two separated rectangles.
- **Merge mid-frame:** U-shape — top row creates two labels that a bottom row joins, forcing an equivalence write.
- **Deep chain:** zigzag pattern forcing chain depth ≥ 4, verifying Phase A path compression.
- **Min-size filter:** one large blob + one 1-pixel speckle, assert only the large blob survives.
- **Overflow:** > `N_LABELS_INT` disjoint tiny blobs; label 0 absorbs surplus, real blobs still emit correctly.
- **Sink discipline:** assert `tready` is held high under all upstream timings.

### Integration — `make run-pipeline`

- `CTRL_FLOW=motion TOLERANCE=0` with the new CCL wired in. Python model and RTL agree pixel-for-pixel on the overlay output.
- Regression on existing synthetic sources: `moving_box`, `two_boxes`, `noisy_moving_box`, `dark_moving_box`.
  - `two_boxes`: two distinct rectangles, visibly separate in the render.
  - `noisy_moving_box`: speckles dropped by `MIN_COMPONENT_PIXELS` filter.
- `CTRL_FLOW=ccl_bbox TOLERANCE=0` on `two_boxes`: two green rectangles on dim-grey mask field.
- `CTRL_FLOW=passthrough` and `CTRL_FLOW=mask` continue to work unchanged (they do not go through `axis_ccl`).
- `make lint` clean.

## Known limitations

1. **Homogeneous-object fragmentation.** Frame differencing fires only on pixel-intensity changes, so a uniform moving object (e.g., a solid-color car) shows only leading/trailing edges in the mask. CCL correctly labels these as *separate* connected components, producing multiple bboxes per visual object. This is a mask-quality limitation inherited from the detector, not a CCL defect. Future fixes, in increasing scope:
   - **(F1)** Morphological closing stage (dilation-then-erosion) between `axis_motion_detect` and `axis_ccl` — bridges small gaps, ineffective once object interior exceeds the dilation kernel.
   - **(F2)** Temporal mask OR — OR the mask over the last *k* frames; different frames contribute different edge positions, painting the interior.
   - **(F3)** Bbox-merge post-pass at CCL output — union output bboxes whose border distance is below a threshold. Cheap (`N_OUT²` combinational pass or small FSM) and orthogonal to CCL's internals. Recommended first future enhancement.

2. **Pathological chain depth.** `MAX_CHAIN_DEPTH = 8` caps Phase A chase; chains deeper than 8 leave tail labels assigned to label 0 (catch-all). Real video produces chains well under 8; adversarial synthetic masks can exceed it. Documented, not fixed.

3. **Internal-label saturation.** If a scene saturates `N_LABELS_INT = 64` mid-frame even with filtering, overflow pixels pool into label 0 and may produce a spurious catch-all bbox. Mitigations: bump `N_LABELS_INT`, or add label recycling (deferred — see Future Work).

4. **No temporal association.** Bboxes from frame N have no identity link to frame N+1. Object tracking (persistent IDs, Kalman/IoU association) is out of scope.

5. **`axis_bbox_reduce` deleted.** If a simplified single-bbox mode is ever needed, it has to be resurrected from git history.

## Future work

- **Bbox-merge post-pass (F3)** — best-value follow-up; addresses the homogeneous-object fragmentation most directly.
- **Cross-frame pipelining** — double-buffer the accumulator and equivalence banks so Phase A–D for frame N runs concurrently with frame N+1's per-pixel labeling. Cost: ~12 Kb distributed RAM (double the per-label state) plus mux logic. Not needed now — real-VGA vblank has ~85× headroom over bounded worst case.
- **Label recycling** — track per-label reference count in the label line buffer; close + free labels that drop to zero. Shrinks `N_LABELS_INT` (e.g., to 16) at the cost of a non-trivial reference-counting FSM. Defer unless saturation is observed in real video.
- **Morphological closing (F1) / temporal mask OR (F2)** — upstream mask-quality improvements that would reduce fragmentation and noise reaching CCL.
